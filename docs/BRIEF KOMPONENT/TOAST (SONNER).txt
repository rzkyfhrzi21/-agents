# BRIEF KOMPONEN: Toast Notification (Sonner)
# Lokasi: Setup di resources/js/Pages/app.tsx + trigger global via Laravel Flash Session
# Tech: sonner + lucide-react + @inertiajs/react

---

## DESKRIPSI

YUBASHOP menggunakan library Sonner untuk toast notification. Notifikasi dapat dipicu langsung secara manual di client-side (React), maupun secara otomatis dikirim dari backend Laravel menggunakan session flash message (`success` / `error`).

---

## SETUP INTEGRASI (resources/js/Pages/app.tsx)

Komponen `<Toaster />` ditempatkan di root app rendering agar siap digunakan kapan saja:

```tsx
import { createRoot } from 'react-dom/client'
import { createInertiaApp } from '@inertiajs/react'
import { Toaster } from 'sonner'

createInertiaApp({
  resolve: name => resolvePageComponent(`./Pages/${name}`, import.meta.glob('./Pages/**/*.tsx')),
  setup({ el, App, props }) {
    createRoot(el).render(
      <>
        <App {...props} />
        <Toaster position="bottom-right" richColors theme="dark" />
      </>
    )
  },
})
```

## TRIGGER TOAST DARI LARAVEL FLASH MESSAGES (Otomatis)

Buat custom hook atau pasang logic di Admin Layout untuk membaca flash message dari backend Laravel secara dinamis:

```tsx
// resources/js/Components/Admin/Layout.tsx
import { usePage } from '@inertiajs/react'
import { useEffect } from 'react'
import { toast } from 'sonner'

export default function Layout({ children }) {
  const { flash } = usePage().props as any

  useEffect(() => {
    if (flash?.success) {
      toast.success(flash.success)
    }
    if (flash?.error) {
      toast.error(flash.error)
    }
  }, [flash])

  return <main>{children}</main>
}
```

### Cara Kirim dari Controller (Laravel):
```php
public function store(Request $request)
{
    // logic simpan...
    return redirect()->route('admin.games.index')
        ->with('success', 'Game baru berhasil ditambahkan!');
}
```

## TRIGGER MANUAL DI CLIENT COMPONENT

```tsx
import { toast } from 'sonner'

// Success Toast
toast.success('Transaksi lunas!')

// Error Toast
toast.error('Gagal memproses data', {
  description: 'Silakan periksa koneksi internet Anda.'
})

// Loading Promise Toast
const promise = saveGameData(payload)
toast.promise(promise, {
  loading: 'Sedang menyimpan game...',
  success: 'Game berhasil disimpan!',
  error: 'Gagal menyimpan game.'
})
```

---

## KETENTUAN PENGGUNAAN TOAST

- **Success:** Digunakan setelah operasi mutasi berhasil (create, update, delete, toggle status).
- **Error:** Digunakan saat query/mutasi gagal, validasi error (jika global), atau kendala jaringan.
- **Warna:** Ditangani otomatis oleh parameter `richColors` (hijau untuk sukses, merah untuk gagal).
