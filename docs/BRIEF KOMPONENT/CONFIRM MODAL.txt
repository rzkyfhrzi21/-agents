## DESKRIPSI

Modal konfirmasi reusable yang muncul sebelum aksi destructive (hapus data, bulk delete, toggle status, dll). Mencegah aksi tidak sengaja.

---

## PROPS INTERFACE

```typescript
interface ConfirmModalProps {
  open: boolean
  onClose: () => void
  onConfirm: () => void | Promise<void>
  title?: string              // default: "Hapus Data"
  description?: string        // default: "Apakah Anda yakin? Tindakan ini tidak dapat dibatalkan."
  confirmText?: string        // default: "Hapus"
  cancelText?: string         // default: "Batal"
  variant?: 'danger' | 'warning'  // default: 'danger'
  isLoading?: boolean         // default: false, disable tombol saat proses
}
```

## CONTOH PEMAKAIAN (Inertia React)

```tsx
// Di Component Page admin games (resources/js/Pages/Admin/Games/Index.tsx)
import { router } from '@inertiajs/react'
import { useState } from 'react'
import { ConfirmModal } from '@/Components/Shared/ConfirmModal'

const [deleteTarget, setDeleteTarget] = useState<Game | null>(null)
const [isDeleting, setIsDeleting] = useState(false)

function handleDelete() {
  if (!deleteTarget) return
  setIsDeleting(true)
  
  router.delete(route('admin.games.destroy', deleteTarget.id), {
    onSuccess: () => {
      setDeleteTarget(null)
      setIsDeleting(false)
    },
    onError: () => {
      setIsDeleting(false)
    }
  })
}

return (
  <>
    <DataTable
      onDelete={(game) => setDeleteTarget(game)}
      ...
    />
    <ConfirmModal
      open={!!deleteTarget}
      onClose={() => setDeleteTarget(null)}
      onConfirm={handleDelete}
      isLoading={isDeleting}
      title="Hapus Game"
      description={`Yakin ingin menghapus "${deleteTarget?.name}"? Semua produk terkait juga akan terhapus.`}
    />
  </>
)
```

---

## CATATAN

- WAJIB dipakai sebelum DELETE — tidak boleh langsung hapus tanpa konfirmasi.
- Mendukung status `isLoading` saat processing request via router Inertia.
- Tutup modal otomatis setelah onConfirm sukses dijalankan.
