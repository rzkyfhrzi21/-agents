## DESKRIPSI

Handler untuk menerima notifikasi webhook dari Midtrans setelah pembeli menyelesaikan pembayaran. Webhook mengupdate status transaksi dan men-trigger notifikasi ke pembeli.

---

## ENDPOINT

POST /api/v1/webhooks/midtrans
- Tanpa middleware `auth` (server-to-server dari Midtrans)
- Validasi via Signature Key verification

---

## ALUR WEBHOOK

```
[Midtrans Server]
  → POST /api/v1/webhooks/midtrans (JSON payload)
  → [WebhookController@midtrans]
    → 1. Log raw payload ke activity_logs
    → 2. Verify Signature Key
    → 3. Find transaction by trx_number (Midtrans order_id = YUBASHOP trx_number)
    → 4. Cek idempotency (sudah SUCCESS? skip)
    → 5. Map Midtrans status → YUBASHOP status
    → 6. Update transaction + simpan raw_response
    → 7. Dispatch notification job
    → 8. Return 200 OK
```

---

## SIGNATURE VERIFICATION

```php
// app/Services/Payment/MidtransWebhookService.php
public function verifySignature(array $payload): bool
{
    $serverKey = config('services.midtrans.server_key');
    $orderId = $payload['order_id']; // Cocok dengan trx_number
    $statusCode = $payload['status_code'];
    $grossAmount = $payload['gross_amount'];

    $expectedSignature = hash('sha512',
        $orderId . $statusCode . $grossAmount . $serverKey
    );

    return $expectedSignature === $payload['signature_key'];
}
```

## STATUS MAPPING

| Midtrans transaction_status | YUBASHOP purchase_status | payment_status |
|-----------------------------|---------------------------|----------------|
| capture (credit card) | Processing | Paid |
| settlement | Processing | Paid |
| pending | Pending | Unpaid |
| deny | Canceled | Unpaid |
| cancel | Canceled | Unpaid |
| expire | Canceled | Unpaid |
| failure | Canceled | Unpaid |

> Catatan: purchase_status jadi "Processing" (bukan "Success") karena top-up masih harus dilakukan manual oleh admin. Admin update ke "Success" setelah top-up manual selesai.

---

## IDEMPOTENCY

Midtrans bisa kirim webhook yang SAMA lebih dari sekali. Cek sebelum update:

```php
if ($transaction->purchase_status === 'Success') {
    // Sudah selesai, jangan proses ulang
    return response()->json(['message' => 'Already processed'], 200);
}
```

---

## ERROR HANDLING

- Selalu return 200 OK (bahkan jika internal error) agar Midtrans tidak retry.
- Wrap semua DB operation dalam try-catch.
- Log error ke activity_logs dan Laravel log.

```php
try {
    $this->webhookService->handle($request->all());
    return response()->json(['message' => 'OK'], 200);
} catch (\Exception $e) {
    Log::error('Midtrans webhook error', [
        'payload' => $request->all(),
        'error' => $e->getMessage(),
    ]);
    return response()->json(['message' => 'OK'], 200);
}
```

## CATATAN

- Controller TIPIS — semua logic di MidtransWebhookService.
- JANGAN proses top-up di webhook — webhook hanya update status.
- Top-up manual oleh admin → admin update status jadi SUCCESS di dashboard.
- Setelah admin update SUCCESS → dispatch job kirim notifikasi WA ke pembeli.
