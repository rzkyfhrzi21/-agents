# BRIEF MODUL: Responsive Mobile
# Project: YUBASHOP
# Stack: Laravel 12 + Inertia React + Tailwind CSS + shadcn/ui
# Target: iPhone 12 (390px) sebagai baseline mobile

---

## DESKRIPSI

Panduan audit mobile responsiveness untuk semua komponen YUBASHOP.
Semua halaman WAJIB 100% mobile-friendly.

---

## 1. DATATABLE (resources/js/Components/Shared/DataTable.tsx)

Tabel dengan banyak kolom di layar kecil = scroll horizontal tidak nyaman.

### Solusi:
- ≥ 768px (md): Tampilan tabel normal
- < 768px: Tetap tabel tapi dengan overflow-x-auto + snap scroll
- Kolom aksi selalu visible (sticky right)

```tsx
<div className="w-full overflow-x-auto">
  <table className="w-full min-w-[600px]">
    {/* ... */}
  </table>
</div>
```

Alternatif (jika tabel terlalu lebar): Card view per baris di mobile
```tsx
{isMobile ? (
  data.map(row => <MobileCardRow key={row.id} row={row} />)
) : (
  <table>...</table>
)}
```

## 2. FORM (resources/js/Components/Modules/*/Form*.tsx)

Semua form layout dua kolom di desktop:
```tsx
<div className="grid grid-cols-1 md:grid-cols-2 gap-6">
```

- Input min-height: min-h-[44px] (Apple HIG touch target)
- Button: full width di mobile (w-full md:w-auto)
- Textarea: min-h-[100px]

---

## 3. MODAL / DIALOG (resources/js/Components/Shared/ConfirmModal.tsx)

- ≥ 768px: Modal centered floating (max-w-md)
- < 768px: Bottom sheet — muncul dari bawah, full width
  ```tsx
  className="fixed inset-x-0 bottom-0 md:inset-auto md:top-1/2 md:left-1/2
             md:-translate-x-1/2 md:-translate-y-1/2 md:max-w-md
             rounded-t-2xl md:rounded-xl"
  ```
- Drag handle bar di atas modal (mobile only): w-10 h-1 bg-slate-600 rounded

---

## 4. KARTU STATISTIK (Dashboard Home)

```tsx
<div className="grid grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 xl:grid-cols-5 gap-4">
  <StatCard ... />
</div>
```

- Mobile (< 640px): 2 kolom
- Tablet (640-1024px): 3 kolom
- Desktop (> 1024px): 4-5 kolom

---

## 5. SIDEBAR (resources/js/Components/Admin/Sidebar.tsx)

- Desktop (≥ 768px): Fixed sidebar 256px
- Mobile (< 768px): Hidden by default, drawer dari kiri
  - Overlay: bg-black/50 fixed inset-0 z-40
  - Drawer: fixed left-0 top-0 h-full w-64 z-50
  - Animasi: translate-x-[-100%] → translate-x-0 transition duration-300
  - Tutup: klik overlay atau klik menu item

---

## 6. HEADER (resources/js/Components/Admin/Header.tsx)

- Desktop: Nama user + avatar + dropdown logout
- Mobile (< 768px):
  - Hamburger button (Menu icon) di kiri
  - Hanya avatar di kanan (nama tersembunyi)
  - Dropdown menu tetap bisa diakses dari avatar

---

## 7. STOREFRONT (Guest Pages)

- Product grid: grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 xl:grid-cols-6
- Product card: aspect-square thumbnail + nama + harga
- Checkout form: single column, full width
- Midtrans Snap popup: otomatis responsive (handled by Midtrans SDK)

## TESTING

1. Chrome DevTools → Toggle Device Toolbar
2. Test di: iPhone 12 (390px), iPad (768px), Desktop (1440px)
3. Cek: touch target ≥ 44px, text readable tanpa zoom, scroll natural
4. Landscape: pastikan tidak ada overflow horizontal

## CATATAN

- Tailwind breakpoints: sm=640px, md=768px, lg=1024px, xl=1280px
- Mobile-first: style default untuk mobile, lalu tambah breakpoint naik
- JANGAN hide fitur di mobile — reorganize, bukan hilangkan
- Test dengan thumb reach zone: tombol penting di bawah layar
