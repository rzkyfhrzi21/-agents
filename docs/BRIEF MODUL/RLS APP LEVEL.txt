# BRIEF MODUL: RLS (Row Level Security) — App Level
# Project: YUBASHOP
# Stack: Laravel 12 + Session Web Middleware + Policy
# BUKAN database-level RLS — implementasi di aplikasi Laravel

---

## DESKRIPSI

Access control di YUBASHOP diimplementasikan di level aplikasi Laravel. Seluruh proteksi halaman admin menggunakan middleware `auth` dan status otorisasi dikirim via Inertia shared props.

---

## ROLE HIERARCHY

| Role | point | Akses |
|------|-------|-------|
| superadmin | 1 | Semua halaman + modul konfigurasi + hapus data |
| admin | 2 | Semua halaman dashboard kecuali modul superadmin-only |
| Guest | - | Tidak login. Checkout, browse catalog, cek pesanan |

---

## MIDDLEWARE

```
app/Http/Middleware/
├── EnsureIsAdmin.php          → role point <= 2 (admin + superadmin)
├── EnsureIsSuperAdmin.php     → role point === 1 (superadmin only)
```

```php
// EnsureIsAdmin.php
public function handle($request, Closure $next)
{
    if (!auth()->check() || auth()->user()->role->point > 2) {
        return redirect()->route('login')->with('error', 'Unauthorized access.');
    }
    return $next($request);
}
```

---

## ROUTE PROTECTION (routes/web.php)

```php
// Guest — tanpa auth
Route::get('/', [HomeController::class, 'index'])->name('home');
Route::post('/transactions', [TransactionController::class, 'store'])->name('transaction.store');
Route::get('/cek-pesanan/{trx_number}', [TransactionController::class, 'checkOrder'])->name('order.check');

// Admin + Superadmin — web session middleware
Route::middleware(['auth', 'is.admin'])->prefix('admin')->name('admin.')->group(function () {
    Route::resource('games', GameController::class);
    Route::resource('products', ProductController::class);
    Route::resource('transactions', TransactionController::class)->except('store');
    Route::resource('users', UserController::class);
    Route::resource('roles', RoleController::class);
    Route::get('logs', [ActivityLogController::class, 'index'])->name('logs');
    Route::resource('config-website', ConfigWebsiteController::class);
});
```

---

## POLICY (Opsional — untuk fine-grained control)

```php
// app/Policies/TransactionPolicy.php
public function delete(User $user, Transaction $trx): bool
{
    return $user->role->point === 1; // superadmin only
}
```

---

## CATATAN

- Guest TIDAK punya record di tabel users.
- Guest cek pesanan via `trx_number` (public route), bukan user session.
- Middleware didaftarkan di `bootstrap/app.php` (Laravel 12).
- Policy hanya untuk kasus spesifik (misal: superadmin boleh delete, admin tidak).
