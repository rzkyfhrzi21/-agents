## 1. N+1 QUERY PROBLEM

Cari semua loop yang di dalamnya ada Eloquent query terpisah.
Ganti dengan eager loading via ->with().

### SEBELUM (buruk):
```php
$transactions = Transaction::paginate(10);
foreach ($transactions as $trx) {
    $product = Product::find($trx->product_id); // N+1!
    $game = Game::find($product->game_id);      // N+1 lagi!
}
```

### SESUDAH (benar):
```php
$transactions = Transaction::with(['product.game'])->paginate(10);
// Hanya 3 query total, bukan N*2+1
```

### FILE YANG PERLU DICEK:
- app/Services/Transaction/TransactionService.php
- app/Services/Product/ProductService.php
- app/Services/AccountStock/AccountStockService.php
- Semua Controller yang memanggil relasi

---

## 2. OVER-FETCHING DATA

Cari query yang pakai `Model::all()` atau `->get()` tanpa select spesifik.
Ganti dengan `->select()` kolom yang dibutuhkan saja.

### SEBELUM:
```php
$games = Game::all(); // ambil SEMUA kolom termasuk desc yang besar
```

### SESUDAH:
```php
$games = Game::select('id', 'name', 'slug', 'img', 'is_popular')->get();
```

### Khusus endpoint list/datatable:
- JANGAN ambil kolom TEXT panjang (desc, content, raw_response)
- Kolom tersebut hanya diambil di endpoint detail (show)

---

## 3. PAGINATION WAJIB

Semua endpoint list HARUS pakai pagination. Tidak boleh ->get() tanpa limit.

```php
// Di Service
public function getAll(int $perPage = 10): LengthAwarePaginator
{
    $perPage = min($perPage, 50); // Maks 50 per halaman
    return Transaction::with('product')
        ->select('id', 'trx_id', 'total_price', 'purchase_status', 'created_at')
        ->latest()
        ->paginate($perPage);
}
```

Default per_page: 10 (sesuai Model $perPage).
Maksimal per_page: 50 (hard limit di controller).

## 4. DATABASE INDEX

Migration untuk index di kolom yang sering di-filter:

```php
// Buat migration terpisah: add_indexes_to_tables
Schema::table('transactions', function (Blueprint $table) {
    $table->index('trx_id');
    $table->index('purchase_status');
    $table->index('payment_status');
    $table->index('product_id');
    $table->index('created_at');
});

Schema::table('products', function (Blueprint $table) {
    $table->index('game_id');
    $table->index('product_code');
});

Schema::table('games', function (Blueprint $table) {
    $table->index('slug');
    $table->index('is_popular');
});

Schema::table('users', function (Blueprint $table) {
    $table->index('email');
    $table->index('role_id');
    $table->index('status');
});

Schema::table('activity_logs', function (Blueprint $table) {
    $table->index('user_id');
    $table->index('created_at');
});
```

## 5. QUERY CACHING (Opsional — Redis)

Untuk data yang jarang berubah (config tables, games list):

```php
use Illuminate\Support\Facades\Cache;

public function getActiveGames()
{
    return Cache::remember('games:active', 3600, function () {
        return Game::where('deleted_at', null)
            ->select('id', 'name', 'slug', 'img')
            ->get();
    });
}
```

Invalidate cache saat data diubah:
```php
Cache::forget('games:active');
```
