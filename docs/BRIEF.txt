## RINGKASAN STRUKTUR PROYEK (DIRECTORY OVERVIEW)

Proyek **YubaStore** adalah aplikasi web top-up game & jual akun game berbasis arsitektur **Monolith (Laravel 12 + Inertia React)**. Berikut adalah ringkasan pembagian folder utama di dalam repositori:

### 1. Folder `backend/` (Monolith Core)
Seluruh kode aplikasi backend dan frontend digabungkan di folder ini.
- `app/` → Berisi logika backend PHP (Eloquent Models, Controllers, Middleware, dan Services seperti Midtrans, Fonnte WA, Resend Email, serta Cloudinary/R2 drivers).
- `config/` → Pengaturan sistem, termasuk `filesystems.php` untuk konfigurasi multi-storage.
- `database/` → Berisi skema migrasi database MySQL (Hostinger) dan database seeder.
- `resources/` → Berisi assets frontend:
  - `resources/js/Pages/` → Halaman-halaman utama Inertia React (Auth, Admin Dashboard, Guest Storefront).
  - `resources/js/Components/` → Komponen UI terpisah (Admin, Guest, dan Shared).
  - `resources/views/app.blade.php` → Kerangka Blade root HTML untuk merender Inertia.
- `routes/` → Kontrol routing web (`routes/web.php`) untuk Inertia, dan api (`routes/api.php`) untuk webhook eksternal.

### 2. Folder `docs/` (Dokumentasi Panduan Proyek)
- `docs/BRIEF MODUL/` → Dokumen panduan spesifik fitur fungsional (Modul Auth, Webhook Midtrans, Proteksi Role, dll.).
- `docs/BRIEF KOMPONENT/` → Standardisasi penulisan komponen UI reusable (DataTable, Form useForm, ConfirmModal, Skeleton Shimmer, Toast Sonner, dll.).
- File kueri spec: `docs/CANVAS.md` (skema tabel database terbaru) dan `docs/ENDPOINT.txt` (alur route & parameter halaman).

### 3. Folder `.ai-skills/` (Panduan Pengerjaan AI Agent)
- Folder konfigurasi berisi instruksi gaya komunikasi, efisiensi pengerjaan kode (YAGNI), serta indeks pembatasan berkas baca untuk AI agent yang berkolaborasi.

## PANDUAN PENGEMBANGAN AI AGENT

Teknologi, versi framework, dan library yang digunakan dalam proyek YubaStore (Laravel 12, Inertia.js, React) memiliki pemutakhiran versi dan konvensi terbaru yang mungkin berbeda dengan basis data pelatihan bawaan model AI Anda.

### Aturan Utama:
1. **Verifikasi API & Konvensi:** Sebelum menulis kode untuk routing, pemanggilan API, atau struktur state Inertia, selalu rujuk dokumentasi resmi terbaru.
2. **Deprecation Notices:** Jangan menggunakan fungsi atau API yang sudah usang (deprecated). Perhatikan warning log atau pesan deprecation pada build tools.
3. **Lazy Senior Dev Mode:** Solusi koding harus se-efisien mungkin, mengutamakan YAGNI (You Aren't Gonna Need It) dan fitur native platform daripada dependensi eksternal tambahan.

## TECH STACK & SYSTEM ARCHITECTURE

| Layer               | Teknologi                                     |
| ------------------- | --------------------------------------------- |
| Frontend Framework  | React (Vite integration inside Laravel)       |
| Monolith Bridge     | Inertia.js (v1.x)                             |
| Styling & UI        | Tailwind CSS + shadcn/ui                      |
| State Management    | Inertia Shared Props (via `usePage().props`)  |
| Tabel Data          | @tanstack/react-table + Shadcn Pagination     |
| Icons & Toast       | lucide-react + sonner                         |
| HTTP Client         | Axios (configured globally with CSRF token)   |
| Object Storage      | Dual-Disk Setup (Cloudinary & Cloudflare R2)  |
| Backend Engine      | Laravel 12 (PHP 8.2+) — Eloquent ORM          |
| Auth System         | Laravel Session & Web Middleware (Stateful)   |
| Database            | MySQL (Hostinger native database)             |
| Cache / Queue       | Upstash Redis (Serverless)                    |
| Payment Gateway     | Midtrans (Snap API + Webhook via Axios/Guzzle)|
| Notifikasi          | Fonnte (WhatsApp) + Resend (Email)            |
| CI/CD               | GitHub Actions                                |
| Aesthetics          | GSAP (GreenSock Animation) for catalog effects|

### Object Storage Disks Setup (`config/filesystems.php`)
```php
'disks' => [
    'local' => [
        'driver' => 'local',
        'root' => storage_path('app'),
        'throw' => false,
    ],

    // DISK 1: CLOUDINARY (Untuk Ikon & Banner Resmi)
    'cloudinary' => [
        'driver' => 'cloudinary',
        'cloud_url' => env('CLOUDINARY_URL'),
    ],

    // DISK 2: CLOUDFLARE R2 (Untuk Galeri Akun & Bukti Joki)
    'r2' => [
        'driver' => 's3',
        'key' => env('CLOUDFLARE_R2_ACCESS_KEY_ID'),
        'secret' => env('CLOUDFLARE_R2_SECRET_ACCESS_KEY'),
        'region' => 'auto',
        'bucket' => env('CLOUDFLARE_R2_BUCKET'),
        'endpoint' => env('CLOUDFLARE_R2_ENDPOINT'),
        'url' => env('CLOUDFLARE_R2_PUBLIC_URL'),
        'use_path_style_endpoint' => true,
    ],
],
```

1. **Cloudinary (Disk 1):** Khusus Ikon & Banner Utama Game
   - **Alasan:** Elemen visual pertama yang dilihat pembeli Guest. Wajib cepat.
   - **Keuntungan:** Auto-Optimization (`f_auto`, `q_auto`) untuk format generasi baru (.webp/.avif).
   - **Efisiensi:** Jumlah terbatas (hanya game aktif), kuota 25 Kredit Gratis bulanan terjamin aman.

2. **Cloudflare R2 (Disk 2):** Khusus Galeri Produk Akun & Bukti Progres Joki
   - **Alasan:** Screenshot spesifikasi akun dan foto pengerjaan joki jumlahnya dinamis dan masif.
   - **Keuntungan:** Flat 10 GB Gratis dengan Free Egress Bandwidth. Menghindari kuota Cloudinary jebol.
   - **Efisiensi:** Cukup akses via invoice ID pembeli, tidak memerlukan kompresi estetis otomatis.

## ROUTING & SECURITY MODEL

### Konsep:
- Routing ditangani sepenuhnya oleh Laravel `routes/web.php`.
- Session Auth (`web` middleware) dan CSRF Protection aktif secara bawaan.
- State dikirim dari Laravel Controller ke React Page via Inertia Shared Props (misalnya: data user login, flash messages, menu navigasi).

### Contoh Proteksi Route (`routes/web.php`):
```php
use App\Http\Controllers\AuthController;
use App\Http\Controllers\Admin\DashboardController;
use App\Http\Controllers\Admin\GameController;
use App\Http\Controllers\Guest\HomeController;

// Guest Routes
Route::get('/', [HomeController::class, 'index'])->name('home');
Route::get('/{slug}', [HomeController::class, 'show'])->name('game.detail');
Route::get('/cek-pesanan', [HomeController::class, 'cekPesanan'])->name('order.check');

// Auth Routes
Route::get('/login', [AuthController::class, 'showLogin'])->name('login')->middleware('guest');
Route::post('/login', [AuthController::class, 'login'])->middleware('guest');
Route::post('/logout', [AuthController::class, 'logout'])->name('logout');

// Protected Admin Routes (Session Web Middleware)
Route::middleware(['auth'])->prefix('admin')->name('admin.')->group(function () {
    Route::get('/', [DashboardController::class, 'index'])->name('dashboard');
    Route::resource('games', GameController::class);
    // Tambahan endpoint CRUD admin lainnya
});
```

## PERSISTENT STATE & SHARED PROPS

State global ditangani oleh Inertia Shared Props untuk mempermudah transfer data yang sering dibutuhkan di frontend tanpa memanggil API tambahan.

### Middleware Sharing (`HandleInertiaRequests.php`):
```php
public function share(Request $request): array
{
    return array_merge(parent::share($request), [
        'auth' => [
            'user' => $request->user() ? [
                'id' => $request->user()->id,
                'name' => $request->user()->name,
                'email' => $request->user()->email,
                'role' => $request->user()->role,
            ] : null,
        ],
        'flash' => [
            'success' => fn () => $request->session()->get('success'),
            'error' => fn () => $request->session()->get('error'),
        ],
    ]);
}
```

### Akses di Frontend (`usePage`):
```tsx
import { usePage } from '@inertiajs/react';

const MyComponent = () => {
  const { auth, flash } = usePage().props;
  // Gunakan auth.user untuk mengecek sesi login
};
```

## MOBILE BEHAVIOR & INTERACTION
- **Sidebar:** Berubah menjadi drawer responsif dengan overlay gelap di mobile, dikendalikan state internal di `resources/js/Components/Admin/Sidebar.tsx`.
- **Tabel:** Overflow horizontal (`overflow-x-auto`) untuk semua tabel `@tanstack/react-table`.
- **Form:** Format kolom tunggal di layar mobile (< 768px), otomatis bertransisi menjadi multi-kolom di desktop.
- **Efek GSAP:** Diaktifkan hanya pada storefront landing page untuk meminimalkan beban rendering di halaman transaksional admin.
